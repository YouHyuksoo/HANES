type: feedback

구현 작업 완료 시 매번 무언가를 빠뜨려서 사용자가 직접 발견하고 지적하는 일이 반복됨.

**Why:** 사용자가 "다음부터는 한번에 처리해라 매번 뭔가를 남겨놓지 말고"라고 피드백. 실제로 menuConfig 누락, DB 테이블 불일치, 시드 데이터 미투입 등이 연달아 발생.

**How to apply:** 구현 완료 보고 전에 반드시 체크리스트 수행:
1. DB 테이블 존재 여부 + 컬럼 일치 검증 (엔티티 vs 실제 DB)
2. 시드 데이터 투입 여부 확인 (실제 DB 조회)
3. 사이드바 menuConfig.ts에 메뉴 등록 여부 확인
4. i18n 4개 파일 키 존재 검증
5. dev 서버 기동 시 에러 없는지 확인
6. 빌드 에러 0건 확인
