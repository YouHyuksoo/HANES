Q6. 워크플로우 정의 방식 — 흐름 구조를 어디에 관리할까요?

7개 워크플로우의 단계, 순서, 분기, 연결 메뉴 경로를 정의하는 방식입니다.

A

프론트엔드 설정 파일 (추천)

config/workflowConfig.ts 에 JSON 형태로 정의. menuConfig.ts와 같은 패턴.
각 워크플로우의 단계(노드), 연결(엣지), 분기 조건, 메뉴 경로를 코드로 관리.
장점: 빠른 구현, 타입 안전, 빌드 타임 검증. 단점: 변경 시 배포 필요.

B

DB 테이블 관리

WORKFLOW_MASTERS, WORKFLOW_STEPS 테이블에 저장. 관리자 화면에서 편집 가능.
장점: 런타임 변경 가능, 관리자가 직접 수정. 단점: 구현 복잡, 워크플로우 편집기 필요.

C

혼합 (설정+DB)

흐름 구조는 프론트엔드 설정 파일, 건수 데이터만 DB/API에서 조회.
워크플로우 자체는 거의 변하지 않으므로 합리적. 나중에 DB로 마이그레이션 가능.
