Q5. 데이터 소스 — 각 노드의 건수/현황을 어디서 가져올까요?

워크플로우 노드에 표시할 실시간 건수의 데이터 소스 전략입니다.

A

정적 설정 (건수 없음)

워크플로우 흐름만 보여주고, 실시간 건수는 표시하지 않음.
순수하게 업무 프로세스 가이드 역할. 빠르게 구현 가능.

B

API 실시간 조회 (추천)

워크플로우 페이지 진입 시 PKG_WORKFLOW Oracle 패키지에서 각 단계별 건수를 한번에 조회.
대시보드 PKG_DASHBOARD와 같은 패턴. 새로고침 버튼으로 갱신.

C

하이브리드 (지연 로딩)

처음엔 흐름만 보여주고, 카드를 호버/클릭하면 해당 워크플로우 건수만 API 조회.
초기 로딩 빠르고, 필요한 데이터만 가져옴.
