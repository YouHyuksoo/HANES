PDA 프로그램 7개 — 확정 워크플로우

2순위 1. 설비 일상점검
시작: 설비 QR 스캔
설비+부품에 바코드 부착
입력: OK/NG 체크
NG시 사유 코드 선택 + 직접입력
NG시: 설비 사용 차단
(인터락 연동)
기타: 정기점검은 PC만
재점검 시 이력 누적 + 안내
1순위 2. 자재입고
순서: 워크플로우 문서 참조
바코드 발행 시 자동 입고
바코드: 제조사 바코드 자동 매핑
매핑 테이블 사전 등록 필수
IQC: 미합격 → 입고 차단
무검사(iqcYn=N) → 바로 입고
취소: PDA 취소 불가
PC에서만 처리
1순위 3. 자재불출 (출고/이동)
시작: 작업지시 바코드 스캔
→ BOM 기반 필요자재 표시
피킹: 불출과 피킹 동시 처리
자재 스캔으로 체크
과불출: 경고 후 허용
"초과입니다. 계속?"
출고유형: 전부 지원
생산불출 / 창고이동 / 반품
반납: 없음 (현장 보관)
1순위 4. 자재재고보정
방식: 자재 바코드 스캔
→ 현재 재고 표시 → 수량 입력
사유: 필수 선택형
(파손/분실/계량오차/기타)
승인: ⚠️ 모든 보정에 관리자 승인 필요 — PDA에서 요청 → PC에서 승인
1순위 5. 자재재고실사
주체: PC에서 시작/종료
PDA는 현장 입력만
방식: 로케이션 스캔
→ 자재 스캔 (자동 카운트)
제한: 실사 중 전체 창고
모든 자재 트랜잭션 차단
확정: PC에서 관리자가
차이 확인 → 보정 확정
1순위 6. 출하실적 등록
순서: 출하지시 스캔
→ 작업자 QR → 제품 스캔(다건)
스캔단위: 개별 박스 +
팔레트 단위 둘 다 지원
검증: 다른 품목 → 차단
초과 스캔 → 차단
부분출하: 허용
취소: PDA 불가, PC만
1순위 7. 제품 재고실사
UI: 자재실사와 동일 구조
(화면 통일)
대상: 완제품만
(반제품 제외)
스캔: 개별 제품 바코드 스캔 (자동 카운트)
