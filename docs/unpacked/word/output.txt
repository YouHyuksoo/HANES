자재 제조사가 부착한 기존 바코드와 MES 내부 바코드의 관계는?

☐ 제조사 바코드 무시 → MES 자체 라벨만 사용
☐ 둘 다 사용 (제조사 바코드로 입고, 이후 MES 라벨 부착)
→ 답변: 바코드 발행 후 MES 라벨과 제조사 바코드 매핑, 매핑 규칙을 설정하는 기준정보 프로그램 필요              (제조사 바코드 : 자리수, 특정 규칙, 포함 단어 등)
WF-03. 자재입고
⚠ 현재 상태: 입고 화면 있으나 바코드 스캔 연동 없음. PDA 미구현. 입고 취소 조건 확정 필요.
Q3-1. 입고 작업 절차는?
자재를 창고에 넣을 때의 단계별 절차를 확인합니다.
☐ IQC 합격 확인 → 자재라벨 스캔 → 창고/로케이션 선택 → 입고 확정
☐ IQC 합격 확인 → 창고/로케이션 바코드 스캔 → 자재 바코드 스캔 → 자동 입고
■ 기타:
라벨 발행 (PC):  
IQC 합격  ->  제조사LOT정보 입력(MES파트넘버와 제조사바코드 규칙 확인)  → 라벨 발행 → 자동 입고 (자재창고, 기본 로케이션)                
자재 입고(PDA)  
창고/로케이션 바코드 스캔 → 자재 바코드 스캔 →  적치
* 이 유사한 형태 진행 예정 : 베트남 자재팀과 협의 확정 예정
Q3-2. 입고 작업 주체는?
☐ 자재 담당자 (PC에서)
☐ 창고 담당자 (PDA로 현장에서)
■ 자재 담당자가 PC에서 등록 + 창고 담당자가 PDA로 적치 확인
→ 답변: _______________________________________________
Q3-3. 로케이션 지정 방식은?

☐ 빈 로케이션에 자유 적치 (적치 시 로케이션 스캔)
☐ 시스템이 로케이션 추천 → 작업자가 확인/변경
→ 답변: _______________________________________________

WF-04. 자재불출 (출고/이동)
⚠ 현재 상태: 작업지시 기반 불출요청 있으나 바코드 리딩 미구현. 자재 분할/병합 상세 로직 미정.
Q4-1. 불출 트리거는?
■ 작업지시 생성 시 BOM 기반 불출요청 자동 생성 → 자재 담당자가 처리
☐ 현장 작업자가 직접 불출 요청 (수동)
☐ 자재 담당자가 작업지시서 확인 후 직접 불출 등록
→ 답변: 필요에 따라 직접 불출도 필요
Q4-2. 불출 작업 절차는?
■ 불출요청서 바코드 스캔 → 자재 바코드 스캔 → 수량 확인 → 불출 확정
☐ 작업지시 선택 → BOM 자재 목록 표시 → 자재 바코드 스캔 → 불출
☐ 기타: _______________
Q4-3. 불출 시 자재 수량 처리는?
⚠ 벌크자재 분할 시 라벨 처리 절차 미정
예: 전선 1,000m 릴에서 작업지시에 필요한 300m만 불출하는 경우
☐ 분할 등록 → 원 시리얼 사용중지 → 700m/300m 두 개의 새 시리얼 생성 → 각각 라벨 발행
☐ 원 시리얼 유지하면서 불출 수량만 차감 (분할 없이)
■ 기타: 분할 가능한 자재는 시리얼 분할, 불가능한 자재(전선 류)는 소요량  혹은 그 이하로 불출 (현장 남은 자재가 있을 수도 있음) 분할 가능 여부는 품목 기준정보에서 설정

■ 작업자가 설비에 자재 장착 전에 자재 바코드 스캔

WF-05. 작업지시
⚠ 현재 상태: 작업지시 생성 구현됨. 완제품 기준 반제품 동시 생성 로직 상세 미정. 트리 형태 현황 미구현.
Q5-1. 작업지시 생성 단위는?
☐ 완제품 기준 1건 생성 → 반제품 작업지시는 BOM 기반 자동 생성
☐ 공정별로 개별 생성 (반제품 단위)
■ 둘 다 가능 (완제품 기준 자동 + 반제품 단독 생성)
→ 답변:  검사공정은 BOM 만으로 지정할수 없음, 라우팅을 같이 사용 해야 함
Q5-2. 작업지시 수량과 실제 생산 수량의 관계는?
■ 작업지시 수량 이상 생산 불가 (초과 시 차단)
☐ 작업지시 수량 초과 생산 가능 (경고만)
☐ 작업지시 수량은 참고용, 실생산 수량은 자유
→ 답변: 1번으로 예상 되지만, 생산팀 재확인 필요
Q5-3. 동일 설비에서 복수 작업지시 동시 진행 가능?
☐ 불가 — 1 설비 = 1 작업지시 (전 작업지시 완료 후 다음 진행)
■ 가능 — 같은 설비에서 여러 작업지시 전환 가능 (혼입 주의)
☐ 같은 품번이면 가능, 다른 품번이면 불가
→ 답변: 같은 설비지만 부품(어플리케이터, 커너날 등 ..) 를 어떤걸 장착 따라 해당 제품 작업이 가능여부가
결정 됨. 작업자가 교체 가능 하며, 그 파트는 품목별 라우팅 공정에서 관리되어야 함 
(작업준비시 설비부품 바코드를 스캔하여 미일치시 인터락)
Q5-4. 작업지시서 출력물에 포함할 정보는?

WF-06. 생산실적 입력 (공통)
⚠ 현재 상태: 리스트 선택 후 키인 입력만 구현. 바코드 없어 실물과 비교 불가. 프로그램 분기 미구현.
Q6-1. 실적 입력 단위는?
⚠ 핵심 질문: 한 번에 1개씩 등록? 모아서 N개 등록? 이에 따라 전체 시스템 구조가 달라짐
☐ 낱개 등록 — 제품 1개 생산할 때마다 즉시 등록 (바코드 스캔)
☐ 묶음 등록 — 일정 수량(예: 10개, 50개) 생산 후 한 번에 등록
☐ 교대 종료 시 일괄 등록 — 교대 동안 생산한 전체 수량을 한 번에
■ 공정별로 다름 (예: 절단은 묶음, 조립은 낱개)
→ 답변: 품목기준정보에 묶음 수량단위 지정 
→ 묶음/일괄인 경우, 묶음 단위는? ___개 또는 ___단위
Q6-2. 생산실적 입력 시 화면 진입 방식은?

☐ 메뉴에서 공정 선택 → 설비 선택 → 입력 화면
☐ 통합 메뉴 → 바코드 유형(설비/자재/제품) 자동 인식 → 화면 분기

Q6-3. 양품/불량 수량 입력 방식은?

■ 생산 수량 입력 → 불량 수량만 별도 입력 → 양품은 자동 계산
☐ 바코드 스캔으로 자동 카운트 (양품 스캔 / 불량은 별도 등록)

→ 답변: 불량 발생시 양품(수량 조정), 불량 바코드 발행

☐ 자재 불출 시점에 이미 매핑됨 → 실적 등록 시 자동 연결
