# 프로젝트 부트스트랩 체크리스트

## 목적

이 문서는 HANES와 유사한 신규 프로젝트를 실제로 시작할 때 필요한 준비 순서를 체크리스트로 정리한 문서다.
AI나 사람이 프로젝트를 구성할 때 빠뜨리기 쉬운 항목을 줄이는 것이 목적이다.

## 1. 시작 전 결정사항

- 프로젝트 도메인 범위를 확정했는가
- 웹과 PDA를 함께 갈지 분리할지 결정했는가
- Oracle을 유지할지 다른 RDBMS로 바꿀지 결정했는가
- 멀티테넌시가 필요한지 결정했는가
- 상태 코드와 공통 코드 체계를 사용할지 결정했는가

## 2. 저장소 구조 준비

- 모노레포 여부를 결정했는가
- `apps/backend`, `apps/frontend` 구조를 사용할지 결정했는가
- 공통 패키지(`shared`) 필요 여부를 결정했는가
- 산출물 폴더를 `exports/`로 분리했는가
- 기준 문서를 `docs/core`로 유지할지 결정했는가

## 3. 백엔드 부트스트랩

- NestJS 프로젝트가 준비됐는가
- TypeORM 연결 구조가 준비됐는가
- DB 환경변수와 사이트 개념이 정의됐는가
- 최상위 도메인 모듈 목록이 정의됐는가
- 상태 전이 메서드 명명 규칙을 정했는가
- 재고, 이력, 로그 테이블 분리 기준을 정했는가

## 4. 프론트엔드 부트스트랩

- Next.js App Router 기준 구조가 준비됐는가
- 웹 영역과 PDA 영역 분리 여부를 정했는가
- 공통 컴포넌트, 훅, API 클라이언트 위치를 정했는가
- 상태값 렌더링을 공통 코드 또는 상태 매퍼로 처리할지 정했는가

## 5. 도메인 설계 체크

- 자재 흐름 정의
- 생산 흐름 정의
- 품질 흐름 정의
- 출하 흐름 정의
- 추적성 흐름 정의
- 취소/복원 흐름 정의

## 6. API 설계 체크

- 도메인 접두사 경로를 정했는가
- 필터 DTO와 생성/수정 DTO를 분리했는가
- 목록 API 페이징 규칙을 정했는가
- 상태 전이 API 경로 규칙을 정했는가
- 에러 응답 메시지 원칙을 정했는가

## 7. 데이터 모델 체크

- 핵심 엔티티 군을 도메인별로 나눴는가
- 자연키, 복합키, 채번키 정책을 정했는가
- 현재 재고와 이력 재고를 분리했는가
- 회사/공장 스코프가 필요한 테이블을 정했는가
- 주요 조회 패턴 기준 인덱스를 설계했는가

## 8. UI 체크

- 목록/상세/입력의 기본 화면 패턴을 정했는가
- PDA 스캔 우선 흐름을 정의했는가
- 공통 필터와 공통 배지 규칙을 정했는가
- 상태 텍스트 하드코딩 금지 규칙을 반영했는가

## 9. 운영 체크

- 스케줄러 필요 여부를 정했는가
- 외부 인터페이스 범위를 정했는가
- 문서/교육/활동로그 같은 운영 모듈 필요 여부를 정했는가
- 빌드, 테스트, 배포 명령을 고정했는가

## 10. AI 주입 체크

- [ai-project-bootstrap.md](C:/Project/HANES/docs/core/ai-project-bootstrap.md)를 먼저 읽혔는가
- 규격 문서 우선 순서대로 주입했는가
- 현재 프로젝트 설명 문서와 규격 문서를 구분했는가
- 안티패턴 문서를 마지막에 다시 읽혔는가
