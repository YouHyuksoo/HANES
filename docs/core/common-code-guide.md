# 공통 코드 가이드

## 목적

공통 코드 시스템을 다른 프로젝트에도 이식할 수 있도록 데이터 구조, API, UI 사용 패턴을 일반화한 문서다.

## 기본 구조

- 그룹 코드
- 상세 코드
- 표시명
- 설명
- 정렬 순서
- 사용 여부
- 부가 속성

## 백엔드 규칙

1. 공통 코드는 별도 엔티티와 API를 가진다.
2. 그룹 목록과 그룹별 상세 목록 API를 분리한다.
3. 활성 코드 전체 조회 API를 제공해 프론트 캐시가 가능하도록 한다.

## 프론트엔드 규칙

1. 상태 배지는 공통 코드 기반 컴포넌트로 그린다.
2. 셀렉트는 공통 코드 전용 컴포넌트를 제공한다.
3. 화면에서 상태명 문자열을 직접 쓰지 않는다.

## 번역 규칙

1. 번역 키가 있으면 번역을 우선한다.
2. 번역이 없으면 DB 표시명을 폴백으로 사용한다.

## 적용 체크리스트

1. 코드 데이터 등록
2. API 노출
3. 프론트 캐시 훅 연결
4. 배지와 셀렉트 컴포넌트 연결
5. 번역 키 추가

## 좋은 예시

- 그룹 목록 API와 그룹별 코드 API를 분리하고, 프론트에서 전체 활성 코드 캐시를 재사용하는 구조
- 상태 표시를 공통 코드 배지 컴포넌트로 통일하는 방식

## 나쁜 예시

- 화면마다 상태 라벨과 색상을 직접 하드코딩하는 방식
- 공통 코드를 만들었지만 번역 키와 폴백 규칙이 없는 방식

## AI 입력 순서에서의 역할

- UI 화면 패턴 뒤에 선택적으로 읽는다.
- 상태, 분류, 배지 체계가 있는 프로젝트에서 바로 사용한다.
