# Exports Policy

`exports/`는 대외 제출용 산출물과 생성 결과물을 두는 폴더다.

## 원칙

1. 이 폴더의 파일은 기준 문서가 아니다.
2. 코드와의 싱크를 보장하지 않는다.
3. 기준 구조, API, 워크플로우 설명은 `docs/`, `docs/reference/`, `docs/diagrams/`를 우선한다.
4. 생성 스크립트 출력은 `exports/`로만 보낸다.

## 구조

- `all/`: 프로젝트 전체 산출물
- `system/`: 시스템 공통 산출물
- `material/`: 자재 도메인 산출물
